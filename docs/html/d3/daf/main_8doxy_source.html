61 \section Overview
62 The class fastuidraw::Painter implements canvas rendering using an implementation
63 of the backend interface fastuidraw::PainterBackend to send data to a 3D API.
64 The class fastuidraw::Painter packs data that is used by multiple triangles
65 into fastuidraw::PainterDrawCommand::m_store (for example the current transformation).
66 In implementing a backend, the shader fed to the GPU needs to correctly unpack
67 this data. The location of the header of the data is stored in the attribute
68 from the value fastuidraw::PainterDrawCommand::m_header_attributes. The header
69 consists of fastuidraw::PainterHeader::header_size uint32_t whose location
70 relative to the start of the header and meaning are enumerated by
71 fastuidraw::PainterHeader::offset_t. The header contains shader ID's
72 and additional offsets to more data packed for a shader to unpack.

219  \section Purpose
220 
221  FastUIDraw provides a high performance, programmable implementation
222  of canvas, found in the class fastuidraw::Painter. FastUIDraw's implementation
223  of canvas reduces the CPU load dramatically and uses a very small
224  number of graphics pipeline states (for example the GL backend
225  using only the features of GL core v3.3, has only 3 pipeline states
226  to implement all of the traditional canvas interface and all 12
227  Porter-Duff blending modes). An API trace of the GL backend of
228  FastUIDraw in action will show that there are only a handful
229  of draw calls per frame (even if there are many different kinds
230  of elements drawn) with very few, if any, pipeline state changes.
231 
232  \section Features
233 
234  The features of fastuidraw::Painter (with the GL backend) are:
235  - Brush (embodied by fastuidraw::PainterBrush)
236  -# linear gradient
237  -# radial gradient
238  -# repeat window
239  -# linear brush tranformation
240  -# image (and repeat image)
241  -# "starting" constant pen color
242  - Drawing primitives
243  -# path stroking
244  -# Ability to change stroking width without recomputation of any data
245  -# Joins
246  - Miter Joins
247  - Bevel Joins
248  - Rounded Joins
249  -# Caps
250  - Square Caps
251  - Rounded Caps
252  - Flat Caps
253  -# Path filling (against arbitary fill rules)
254  -# GPU rendered glyphs
255  -# Quads
256  -# Ability for application to have custom drawing types via shaders
257  - Clipping
258  -# ClipIn against filled paths
259  -# ClipOut against filled paths
260  - Blend Modes
261  -# All Porter-Duff blend modes
262  -# Ability for application to have custom blending types via shaders
263 
264  \section Concepts
265  FastUIDraw strongly distinguishes between "what" to draw and "how" to draw.
266  The "what" to draw is embodied by attribute and index data. This data is
267  expected to be reused and, for the fastuidraw::Painter interface, is embodied
268  by the fastuidraw::PainterAttributeData. The "how" to draw is small amounts
269  of data that includes transformation state, brush state and so on. For the class
270  fastuidraw::Painter, the "how" to draw is embodied by fastuidraw::PainterData.
271  The expectation is "how" to draw can be changed without incurring significant
272  CPU load.
273 
274  \section Resources
275  All of the "resources" used by a canvas are to reside on a set
276  of atlases:
277  - Images reside on a fastuidraw::ImageAtlas (see also fastuidraw::Image)
278  - Glyphs reside on a fastuidraw::GlyphAtlas (see also fastuidraw::Glyph)
279  - Color stops reside on a fastuidraw::ColorStopAtlas (see also fastuidraw::ColorStopSequenceOnAtlas)
280  An instance of fastuidraw::Painter has that it sources all its resources from a fixed
281  set of atlases.
282 
283  \section Text
284  The elements in the group \ref Text are for the realization of glyphs. FastUIDraw
285  does <i>NOT</i> provide functions for layout of glyphs. The API exposed through
286  fastuidraw::GlyphSequence is that an application provides a sequence of glyphs,
287  glyph positions and the the size at which the text has been layed out. Glyph objects
288  reside on a fastuidraw::GlyphCache which provides an interface to fetch glyphs from
289  a glyph code and a fastuidraw::GlyphRender. A fastuidraw::GlyphRender
290  specifies the type of resource data on how to render glyphs. The class
291  fastuidraw::GlyphSelector provides assistance for selection of glyphs from
292  character codes, but an application can use any system it likes for glyph selection.
293  FastUIDraw has the class fastuidraw::FontFreeType to construct a font via Freetype
294  (either from a file or FT_Face). Below is an example of startup code and simple layout code.
295  \code
296  //startup
297  // choose an orientation convention
298  // # y_increases_downwards --> y = 0 is at top of the window and increases down the window
299  // # y_increases_upwards --> y = 0 is at bottom of the window and increases up the window
300  enum fastuidraw::PainterEnums::screen_orientation myOrientation(PainterEnums::y_increases_downwards);
301 
302  //use GL backend atlas
303  fastuidraw::reference_counted_ptr<fastuidraw::GlyphAtlas> myGlyphAtlas;
304  myGlyphAtlas = FASTUIDRAWnew fastuidraw::gl::GlyphAtlasGL(fastuidraw::gl::GlyphAtlasGL::params());
305 
306  //setup glyph cache and glyph selector; the glyph selector
307  //is a VERY simple class to enable finding glyphs from fonts
308  //and falling back to other fonts when a glyph cannot be found
309  fastuidraw::reference_counted_ptr<fastuidraw::GlyphCache> myGlyphCache;
310  fastuidraw::reference_counted_ptr<fastuidraw::GlyphSelector> myGlyphSelector;
311  myGlyphCache = FASTUIDRAWnew fastuidraw::GlyphCache(myGlyphAtlas);
312  myGlyphSelector = FASTUIDRAWnew fastuidraw::GlyphSelector();
313 
314  //create a generator object that the font uses to load the font data,
315  //in this example we load the font contents from a file directly.
316  fastuidraw::reference_counted_ptr<fastuidraw::FreeTypeFace::GeneratorBase> gen;
317  gen = FASTUIDRAWnew fastuidraw::FreeTypeFace::GeneratorFile("myFont.ttf", 0);
318 
319  //create the font
320  myFont = FASTUIDRAWnew fastuidraw::FontFreeType(gen);
321 
322  //add font to GlyphSelector
323  myGlyphSelector->add_font(font);
324 
325  //creating glyph sequence.
326  const char *text = "Hello World";
327  float layout_pixel_size(32.0f); //a pixel size by which to layout text
328  std::vector<fastuidraw::Glyph> glyphs;
329  std::vector<fastuidraw::vec2> glyph_positions;
330  fastuidraw::vec2 pen_position(0.0f, 0.0f);
331  for(const char *p = text; *p; ++p)
332  {

334  g = myGlyphSelector->fetch_glyph(myFont, *p);
335  if(g.m_font)
336  {

338  glyph_positions.push_back(pen_positions);

356  - The above layout is very limited, but because an application can
357  construct fonts from FT_Face objects, one can use text shaping
358  and layout libraries of one's choice to perform layout.

360  \section CustomDrawing Custom Drawing
361  In addition to drawing glyph sequences, rectangles, filling and stroking paths,
362  FastUIDraw provides an interface to applications to create attribute and data as it sees fit.
363  For drawing of an item there are three kinds of data used to draw an item.
364  -# Attribute data which is value per vertex emitted. Attribute data is realized by
365  fastuidraw::PainterAttribute values.
366  -# Index data is indices into an array of fastuidraw::PainterAttribute to specify what
367  triangles to draw. Indices are realized as fastuidraw::PainterIndex values.
368  -# Reused data is data that is common to all vertices and pixels of an item or even
369  multiple items. This data is realized by fastuidraw::generic_data. This data
370  is realized as part of fastuidraw::PainterData::m_item_shader_data.
371 
372  To draw custom data, use the function fastuidraw::Painter::draw_generic().
373  The object fastuidraw::PainterItemShader encapsulates how to process vertices and
374  fragments for the draw. A custom shader is tied to the backend.
375  However, these classes are only the interface for fastuidraw::Painter.
376  The actual shading code is tied to the GPU API one is using. For GL (and GLES)
377  backend, the shader class is the class fastuidraw::glsl::PainterItemShaderGLSL.
378  In that class, one writes GLSL code. WARNING: if one write incorrect GLSL code,
379  the uber-shader will fail to compile and all painting is then disabled!
380 
381  All pixels are passed through the brush of fastuidraw::Painter,
382  embodied by fastuidraw::PainterBrush. That brush class contains the abiltiy
383  to apply gradients (linear, radial), repeat windowing, transformation of
384  brush image, pen color and image (complete with repeating and filtering the
385  image).

496  interface to create a backend. A backend implementation encompasses
497  creating backend classes for the following base interface classes
498  - fastuidraw::ImageAtlas (the GL backend class is given by fastuidraw::gl::ImageAtlasGL)
499  - fastuidraw::ColorStopAtlas (the GL backend class is given by fastuidraw::gl::ColorStopAtlasGL)
500  - fastuidraw::GlyphAtlas (the GL backend class is given by fastuidraw::gl::GlyphAtlasGL)

502  - fastuidraw::PainterBlendShader (the GL backend class is given by fastuidraw::glsl::PainterBlendShaderGLSL)
503  - fastuidraw::PainterItemShader (the GL backend class is given by fastuidraw::glsl::PainterItemShaderGLSL)
