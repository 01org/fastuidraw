3  \brief Theory and Concept in FastUIDraw
4  \section Concepts Concepts
5  FastUIDraw strongly distinguishes between "what" to draw and "how" to draw.
6  The "what" to draw is embodied by attribute and index data. This data is
7  expected to be reused and, for the fastuidraw::Painter interface, is embodied
8  by the fastuidraw::PainterAttributeData. The "how" to draw is small amounts
9  of data that includes transformation state, brush state and so on. For the class
10  fastuidraw::Painter, the "how" to draw is embodied by fastuidraw::PainterData.
11  The expectation is "how" to draw can be changed without incurring significant
12  CPU load.
13 
14  \section CustomDrawing Custom Drawing
15  In addition to drawing glyph sequences, rectangles, filling and stroking paths,
16  FastUIDraw provides an interface to applications to create attribute and data as it sees fit.
17  For drawing of an item there are three kinds of data used to draw an item.
18  -# Attribute data which is value per vertex emitted. Attribute data is realized by
19  fastuidraw::PainterAttribute values.
20  -# Index data is indices into an array of fastuidraw::PainterAttribute to specify what
21  triangles to draw. Indices are realized as fastuidraw::PainterIndex values.
22  -# Reused data is data that is common to all vertices and pixels of an item or even
23  multiple items. This data is realized by fastuidraw::generic_data. This data
24  is realized as part of fastuidraw::PainterData::m_item_shader_data.
25 
26  To draw custom data, use the function fastuidraw::Painter::draw_generic().
27  The object fastuidraw::PainterItemShader encapsulates how to process vertices and
28  fragments for the draw. A custom shader is tied to the backend.
29  However, these classes are only the interface for fastuidraw::Painter.
30  The actual shading code is tied to the GPU API one is using. For GL (and GLES)
31  backend, the shader class is the class fastuidraw::glsl::PainterItemShaderGLSL.
32  In that class, one writes GLSL code. WARNING: if one write incorrect GLSL code,
33  the uber-shader will fail to compile and all painting is then disabled!
34 
35  All pixels are passed through the brush of fastuidraw::Painter,
36  embodied by fastuidraw::PainterBrush. That brush class contains the abiltiy
37  to apply gradients (linear, radial), repeat windowing, transformation of
38  brush image, pen color and image (complete with repeating and filtering the
39  image).
