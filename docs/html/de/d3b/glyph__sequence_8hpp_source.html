38  /*!
39  * \brief
40  * A GlyphSequence represents a sequence of glyph codes with positions.
41  * A GlyphSequence provides an interface to grab the glyph-codes realized
42  * as different renderers for the purpose of rendering text in response
43  * to the transformation that a Painter currently has. A GlyphSequence
44  * maintains a hierarchy so that Painter can quickly cull glyphs that
45  * are not visible. The methods of GlyphSequence are re-entrant but not
46  * thread safe, i.e. if an application uses the same GlyphSequence from
47  * multiple threads it needs to explicitely handle locking itself when
48  * using it.
49  */
