32  /*!
33  * A GlyphRenderDataRestrictedRays represents the data needed to
34  * build a glyph to render it with a modification to the technique
35  * of "GPU-Centered Font Rendering Directly from Glyph Outlines"
36  * by Eric Lengyel. The modifications we have to the technique are
37  * as follows.
38  * - We break the glyph's box into a hierarchy of boxes where
39  * each leaf node has a list of what curves are in the box
40  * together with a single sample point inside the box giving
41  * the winding number at the sample point.
42  * - To compute the winding number, one runs the technique
43  * on the ray connecting the fragment position to the winding
44  * sample position and increment the value by the winding value
45  * of the sample. Here the main caveat is that one needs to
46  * ignore any intersection that are not between the fragment
47  * position and the sample position.
48  * - The shader (which can be fetched with the function
49  * fastuidraw::glsl::restricted_rays_compute_coverage())
50  * tracks the closest curve (in a local L1-metric scaled to
51  * window coordinates) to the fragment position that increments
52  * the winding value and also tracks the closest curve that
53  * decrements the winding value. Using those two values together
54  * with the winding value allows the shader to compute a coverage
55  * value to perform anti-aliasing.
56  */
