31  /*!
32  * A GlyphRenderDataRestrictedRays represents the data needed to
33  * build a glyph to render it with a modification to the technique
34  * of "GPU-Centered Font Rendering Directly from Glyph Outlines"
35  * by Eric Lengyel. The modifications we have to the technique are
36  * as follows.
37  * - We break the glyph's box into a hierarchy of boxes where
38  * each leaf node has a list of what curves are in the box
39  * together with a single sample point inside the box giving
40  * the winding number at the sample point.
41  * - To compute the winding number, one runs the technique
42  * on the ray connecting the fragment position to the winding
43  * sample position and increment the value by the winding value
44  * of the sample. Here the main caveat is that one needs to
45  * ignore any intersection that are not between the fragment
46  * position and the sample position.
47  * - The shader (which can be fetched with the function
48  * fastuidraw::glsl::restricted_rays_compute_coverage())
49  * tracks the closest curve (in a local L1-metric scaled to
50  * window coordinates) to the fragment position that increments
51  * the winding value and also tracks the closest curve that
52  * decrements the winding value. Using those two values together
53  * with the winding value allows the shader to compute a coverage
54  * value to perform anti-aliasing.
55  */
