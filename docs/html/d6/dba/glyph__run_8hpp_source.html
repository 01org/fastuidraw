38  /*!\brief
39  * A GlyphRun represents a sequence of glyph codes with positions.
40  * A GlyphRun provides an interface to grab the glyph-codes realized
41  * as different renderers for the purpose of rendering text in response
42  * to the transformation that a Painter currently has. In contrast to
43  * a \ref GlyphSequence, a GlyphRun does NOT maintain a hierarchy
44  * to perform culling. However, it does provide an interface to select
45  * continuous subsets of the glyphs for the purporse of rendering.
46  * In addition, since it does not carry a hierarchy for culling, it
47  * is also a lighter weight object than \ref GlyphSequence. Like \ref
48  * GlyphSequence, the methods of \ref GlyphRun are re-entrant but not
49  * thread safe, i.e. if an application uses the same \ref GlyphRun from
50  * multiple threads it needs to explicitely handle locking itself when
51  * using it.
52  */
