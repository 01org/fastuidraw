33  /*!
34  * \brief
35  * An ImageAtlasGL is the GL(and GLES) backend implementation
36  * for \ref ImageAtlas.
37  *
38  * An ImageAtlasGL on creation, creates an
39  * \ref AtlasColorBackingStoreBase and an \ref AtlasIndexBackingStoreBase
40  * itself that are backed by GL textures. Both of the backing
41  * stores use GL_TEXTURE_2D_ARRAY textures. On deletion,
42  * deletes the backing color and index stores.
43  *
44  * The method flush() must be called with a GL context current.
45  * If the ImageAtlasGL was constructed as delayed,
46  * then the loading of data to the GL textures is delayed until
47  * flush(), otherwise it is done immediately and then must be done
48  * with a GL context current.
