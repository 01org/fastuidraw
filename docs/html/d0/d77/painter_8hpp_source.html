59  /*!
60  * \brief
61  * Painter implements a canvas rendering interface.
62  *
63  * Painter implements:
64  * - stroking
65  * - filling
66  * - drawing text
67  * - applying a brush (see \ref PainterBrush)
68  * - blending (see \ref PainterEnums::blend_mode_t)
69  * - single 3x3 transformation
70  * - save and restore state
71  * - transparency layers
72  * - clipIn against Path, rectangle or rounded rectangle
73  * - clipOut against Path, rectangle or rounded rectangle
74  *
75  * The transformation of a Painter goes from local item coordinate
76  * to 3D API clip-coordinates (for example in GL, from item coordinates
77  * to gl_Position.xyw). FastUIDraw follows the convention that the top
78  * of the window is at normalized y-coordinate -1 and the bottom of the
79  * window is at normalized y-coordinate +1. The transformation is to be
80  * applied as matrix-vector multiplication, i.e.
81  * \code
82  * ClipCoordinates = transformation() * vec3(x, y, 1.0)
83  * \endcode
84  * for local coordiante (x, y). Normalized device coordinates are
85  * defined as
86  * \code
87  * NormalizedDeviceCoordinates = ClipCoordinates.xy / ClipCoordinates.w
88  * \endcode
89  * where (-1, -1) corresponds to the bottom-left hand corner of the
90  * viewport (see PainterSurface::viewport()) and (+1, +1)
91  * is the top right hand corner of the viewport.
92  *
93  * The pixel pipeline of Painter is
94  * - Compute RGBA value from item shader (typically this is (0, 0, 0, alpha)
95  * where alpha is a coverage value
96  * - Modulate by the PainterBrush passing the item coordinates
97  * of the pixel to the \ref PainterBrush
98  * - Apply blending (see PainterEnums::blend_mode_t and
99  * Painter::blend_shader()) to the RGBA value against the
100  * current value in the framebuffer
101  *
102  * Painter uses clip-planes and the depth buffer to perform clipping.
103  * The depth-buffer clips by occluding elements. For example, the
104  * method clip_out_rect() simply draws a rectangle so that it does
105  * not affect the color buffer but with a depth value that is infront
106  * of the elements that it is to occlude. Painter uses the convention
107  * that elements with greater than or equal depth values are visible
108  * (for example in GL this corresponds to the depth test being set to
109  * GL_GEQUAL). Unless an item occludes (or self occludes), the current
110  * active depth value is unaffected. An item's vertex shader will emit
111  * a relative z-value (i.e. relative to the item) which is then
112  * incremented by the current z-value of the \ref Painter.
113  *
114  * A Painter is a HEAVY object (it creates pools of buffers during its
115  * lifetime for reuse). As such, one should reuse Painter objects as
116  * much as reasonably possible.
117  */
