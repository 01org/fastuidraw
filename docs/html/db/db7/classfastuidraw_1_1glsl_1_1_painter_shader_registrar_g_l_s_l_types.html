enum fastuidraw::glsl::PainterShaderRegistrarGLSLTypes::auxiliary_buffer_t

Enumeration to describe auxiliary buffer support

Enumerator
no_auxiliary_buffer 

No auxiliary buffer is present

auxiliary_buffer_atomic 

Auxiliary buffer is realized by atomic operations; To guarantee ordering between computing coverage and using coverage a backend must intert a memory barrier at the API level between such passed (for example in GL, glMemoryBarrier()).The buffer is realized as an "r32ui" image2D in the shader source.

auxiliary_buffer_interlock 

Auxiliary buffer is present and ordering guarantees are implemented by an interlock that can be called in GLSL from any function and/or control flow. This allows for cover then draw methods to be performed WITHOUT any draw-breaks. The buffer is realized as an "r8" image2D in the shader source.

auxiliary_buffer_interlock_main_only 

Auxiliary buffer is present and ordering guarantees are implemented by an interlock that can only be called in GLSL from main under NO conrol flow. This allows for cover then draw methods to be performed WITHOUT any draw-breaks. The buffer is realized as an "r8" image2D in the shader source.

auxiliary_buffer_framebuffer_fetch 

Auxiliary buffer is present and ordering guarantees are implemented via framebuffer fetch, i.e. the auxiliary buffer is present as an inout global with type float and layout(location = 1).

enum fastuidraw::glsl::PainterShaderRegistrarGLSLTypes::clipping_type_t

Enumeration specifying how the uber-shaders will perform clipping against the clip-planes of PainterClipEquations

Enumerator
clipping_via_gl_clip_distance 

Clipping is performed in the vertex-shader using gl_ClipDistance[i] for 0 <= i < 4.

clipping_via_discard 

Clipping is performed by passing the distance to each clip-plane and performing discard in the fragment shader.

clipping_via_skip_color_write 

Clipping is performed by passing the distance to each clip-plane and (virtually) skipping the color write. This requires that the compositing mode is through framebuffer fetch, i.e. UberShaderParams::compositing_type() is compositing_framebuffer_fetch or compositing_interlock

enum fastuidraw::glsl::PainterShaderRegistrarGLSLTypes::compositing_type_t

Enumeration to specify how to perform painter compositing.

Enumerator
compositing_single_src 

Use single source compositing; only compositing is supported and blending is not supported.

compositing_dual_src 

Use dual soruce compositing; only compositing is supported and blending is not supported.

compositing_framebuffer_fetch 

Use framebuffer fetch compositing; both compositing and blending are supported.

compositing_interlock 

Have the color buffer realized as an image2D and use fragment shader interlock to get compositing order correct; compositing and blending are supported.
