35  /*!
36  * \brief
37  * A PainterBrushShaderGLSL is a PainterBrushShader whose
38  * shader code fragment is via GLSL.The vertex shader code needs to implement the function:
39  * \code
40  * void
41  * fastuidraw_gl_vert_brush_main(in uint sub_shader,
42  * inout uint shader_data_block,
43  * in vec2 brush_p)
44  * \endcode
45  * where
46  * - sub_shader corresponds to PainterBrushShader::sub_shader()
47  * - brush_p is the brush position emitted by the item shader and
48  * - shader_data_block is what block in the data store for
49  * the data packed by PainterBrushShaderData::pack_data()
50  * of the PainterBrushShaderData in the \ref Painter call;
51  * use the macro fastuidraw_fetch_data() to read the data.
52  *
53  * The fragment shader code needs to implement the function:
54  * \code
55  * vec4
56  * fastuidraw_gl_frag_brush_main(in uint sub_shader,
57  * inout uint shader_data_block)
58  * \endcode
59  * which returns the color value, pre-multiplied by alpha, by which
60  * to modulate the outgoing fragment color.
61  *
62  * Available to only the vertex shader are the following:
63  * - the GLSL elements in the module \ref GLSLVertCode
64  *
65  * Available to only the fragment shader are the following:
66  * - the GLSL elements in the module \ref GLSLFragCode
67  *
68  * Available to both the vertex and fragment shader are the following:
69  * - the GLSL elements in the module \ref GLSLVertFragCode
70  *
71  * For both stages, the value of the argument of shader_data_block is
72  * which 128-bit block into the data store (PainterDraw::m_store) of the
73  * shader data to be read with the GLSL macro \ref fastuidraw_fetch_data.
74  * On exit, this value must be updated to the location just past the
75  * shader data of the shader.
76  *
77  * For both stages, the value of the argument of sub_shader() is the
78  * value of \ref PainterShader::sub_shader() of the active shader.
79  *
80  * Also, if one defines macros in any of the passed ShaderSource objects,
81  * those macros MUST be undefined at the end. In addition, if one
82  * has local helper functions, to avoid global name collision, those
83  * function names should be wrapped in the macro FASTUIDRAW_LOCAL()
84  * to make sure that the function is given a unique global name within
85  * the uber-shader.
86  *
87  * Lastly, one can use the class \ref UnpackSourceGenerator to generate
88  * shader code to unpack values from the data in the data store buffer.
89  * That machine generated code uses the macro fastuidraw_fetch_data().
90  */
91  class PainterBrushShaderGLSL:public PainterBrushShader

94  /*!
95  * \brief
96  * If one wishes to make use of other \ref PainterBrushShaderGLSL
97  * fastuidraw_gl_vert_brush_main()/fastuidraw_gl_frag_brush_main()
98  * of other shaders (for example to have a simple shader that adds
99  * on to a previous shader), a DependencyList provides the means to do so.
100  *
101  * Each such used shader is given a name by which the caller will use it.
102  * In addition, the caller has access to the varyings and shared symbols
103  * of the callee as well. A varying or shareable V of an element in the \ref
104  * DependencyList is accessed from the parent shader with dep::V where dep
105  * is the argument value of name to \ref add_shader(). Note that it is
106  * accessed with the scope-resolution operator; the uber-shader assember
107  * will convert the scope-resolution operator into acceptable GLSL code.
108  *
109  * By using the values of the shareables (embodied in fields \ref
110  * symbol_list::m_vert_shareable_values and \ref symbol_list::m_frag_shareable_values),
111  * reading and potentially modifying the values of the varyings, one can
112  * create effects building off of the built-in shaders of the GLSL module.
113  *
114  * Lastly, the uber-shader assembler gives a means to avoid collision
115  * in using context textures. To avoid re-using the same context
116  * textures, the macros fastuidraw_brush_start_context_texture,
117  * and fastuidraw_brush_context_texture(X) are provided where
118  * - fastuidraw_brush_start_context_texture is the first texture for
119  * the brush shader to use.
120  * - fastuidraw_brush_context_texture(X) is just
121  * fastuidraw_brush_context_texture[X + fastuidraw_brush_start_context_texture]
122  *
123  * The Uber-shader assembler fills the textures depth-first. Thus the first
124  * dependency uses slots [0, 1, .., N - 1] where N is number_context_textures()
125  * of the first dependency, then the second dependency uses [N, N + 1, ..., N + M - 1]
126  * where M is number_context_textures() of the second dependency and so on.
127  */
