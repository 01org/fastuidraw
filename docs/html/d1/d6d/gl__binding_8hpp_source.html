32 /*!
33  * \brief
34  * Provides interface for application to use GL where function pointers
35  * are auto-resolved transparently and under debug provides error checking.
36  * Built as a part of a seperate library; for GL it is libNGL; for GLES
37  * it is NGLES. The header defines for each GL/GLES function, glFoo, the
38  * macro fastuidraw_glFoo.
39  *
40  * Short version:
41  * - application should call fastuidraw::gl_binding::get_proc_function()
42  * to set the function which will be used to fetch GL function pointers.
43  * - If an application wishes, it can include <fastuidraw/gl_backend/ngl_header.hpp>.
44  * The header will add the GL function-macros and an application can
45  * issue EGL calls without needing to fetch the GL functions via
46  * fastuidraw_glFoo where glFoo is the EL funcion to all. Under release
47  * the macros are defined to function pointers that automatically set
48  * themselves up correcty. For debug, the macros preceed and postceed each
49  * GL function call with error checking call backs so an application writer
50  * can quickly know what line/file triggered an GL error. If an application does
51  * not wish to use the macro system (and will also need to fetch function pointers
52  * itself) it can just include <fastuidraw/gl_backend/gl_header.hpp> which
53  * will include the correct system GL/GLES headers.
54  *
55  * Long Version:
56  *
57  * The namespace gl_binding provides an interface for an application
58  * to specify how to fetch GL function pointers (see
59  * fastuidraw::gl_binding::get_proc_function()) and additional
60  * functionality of where to write/store GL error messages. An application
61  * can also use this functionality by including <fastuidraw/ngl_gl.hpp>.
62  * The header will create a macro fastuidraw_glFoo for each GL function
63  * glFoo. If FASTUIDRAW_DEBUG is defined, each GL call will be preceded
64  * by a callback and postceeded by another call back. The preceed callback
65  * to the GL call will call the implementation of CallbackGL::pre_call() of
66  * each active CallbackGL object. The post-process callback will repeatedly
67  * call glGetError (until it returns no error) to build an error-string. If
68  * the error string is non-empty, it is printed to stderr. In addition,
69  * regardless if the error-string is non-empty, CallbackGL::post_call() of
70  * each active CallbackGL is called.
71  *
72  * To fetch the function pointer of a GL function,
73  * use the macro <B>FASTUIDRAWglfunctionPointer</B> together
74  * with <B>FASTUIDRAWglfunctionExists</B>. The macro
75  * <B>FASTUIDRAWglfunctionPointer</B> will NEVER return
76  * a nullptr pointer. For the cases where the GL implementation
77  * does not have that function, the function pointer
78  * returned will then point to a do-nothing function.
79  * To check if a GL implementation has a given function
80  * use the macro-function <B>FASTUIDRAWglfunctionExists</B>
81  * which returns non-zero if the GL implementation has
82  * the function. Some example code:
83  *
84  * \code
85  * //get a function pointer for a GL function
86  * //which takes no arguments and returns nothing.
87  * void (*functionPointer)(void) = nullptr;
88  * if (FASTUIDRAWglfunctionExists(glSomething)
89  * {
90  * functionPointer = FASTUIDRAWglfunctionPointer(glSomeFunction);
91  * }
92  * else
93  * {
94  * //FASTUIDRAWglfunctionPointer(glSomeFunction) is NOT nullptr,
95  * //rather it maps to a no-op function.
96  * //in this example we leave the value of
97  * //functionPointer as nullptr to indicate the function
98  * //is not supported by the GL implementation.
99  * }
100  * \endcode
101  *
102  * Calling a GL function through a function pointer
103  * will bypass the GL error checking ad callbacks though.
104  * One issue with using FASTUIDRAWglfunctionExists is that a
105  * number of GL implementations will return a function pointer,
106  * even if the implementation does not support it. As always,
107  * when fetching function pointers, one should check the GL
108  * version and GL extension string(s) to know if the GL
109  * implementation supports that function.
110  *
111  * The gl_binding system requires that an application provides
112  * a function which the binding system uses to fetch function
113  * pointers for the GL API, this is set via
114  * gl_binding::get_proc_function().
115  */
