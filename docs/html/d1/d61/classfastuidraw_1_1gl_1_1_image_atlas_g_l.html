Detailed Description

An ImageAtlasGL is the GL(and GLES) backend implementation for ImageAtlas.

An ImageAtlasGL on creation, creates an AtlasColorBackingStoreBase and an AtlasIndexBackingStoreBase itself that are backed by GL textures. Both of the backing stores use GL_TEXTURE_2D_ARRAY textures. On deletion, deletes the backing color and index stores.

The method flush() must be called with a GL context current. If the ImageAtlasGL was constructed as delayed, then the loading of data to the GL textures is delayed until flush(), otherwise it is done immediately and then must be done with a GL context current.
