344  /*!
345  * \brief
346  * A PainterItemShaderGLSL is a collection of GLSL source code
347  * fragments for a \ref PainterShaderRegistrarGLSL.
348  *
349  * The vertex shader code needs to implement the function:
350  * \code
351  * void
352  * fastuidraw_gl_vert_main(in uint sub_shader,
353  * in uvec4 attrib0,
354  * in uvec4 attrib1,
355  * in uvec4 attrib2,
356  * in uint shader_data_offset,
357  * out uint z_add,
358  * out vec2 brush_p,
359  * out vec3 clip_p)
360  * \endcode
361  * where
362  * - sub_shader corresponds to PainterItemShader::sub_shader()
363  * - attrib0 corresponds to PainterAttribute::m_attrib0,
364  * - attrib1 corresponds to PainterAttribute::m_attrib1,
365  * - attrib2 corresponds to PainterAttribute::m_attrib2 and
366  * - shader_data_offset is what block in the data store for
367  * the data packed by PainterItemShaderData::pack_data()
368  * of the PainterItemShaderData in the \ref Painter (or
369  * \ref PainterPacker) call; use the macro fastuidraw_fetch_data()
370  * to read the data.
371  *
372  * The output clip_p is to hold the clip-coordinate of the vertex.
373  * The output brush_p is to hold the coordinate for the brush of
374  * the vertex. The out z_add must be written to as well and it
375  * is how much to add to the value in \ref PainterHeader::m_z
376  * (the value is the value of Painter::current_z()) for the purpose
377  * of intra-item z-occluding.
378  *
379  * The fragment shader code needs to implement the function:
380  * \code
381  * vec4
382  * fastuidraw_gl_frag_main(in uint sub_shader,
383  * in uint shader_data_offset)
384  * \endcode
385  *
386  * which returns the color of the fragment for the item -before-
387  * the color modulation by the pen, brush or having compositing
388  * applied. In addition, the color value returned is NOT
389  * pre-multiplied by alpha either.
390  *
391  * Available to only the vertex shader are the following:
392  * - mat3 fastuidraw_item_matrix (the 3x3 matrix from item coordinate to clip coordinates)
393  *
394  * Available to both the vertex and fragment shader are the following:
395  * - sampler2DArray fastuidraw_imageAtlasLinear the color texels (AtlasColorBackingStoreBase) for images unfiltered
396  * - sampler2DArray fastuidraw_imageAtlasLinearFiltered the color texels (AtlasColorBackingStoreBase) for images bilinearly filtered
397  * - usampler2DArray fastuidraw_imageIndexAtlas the texels of the index atlas (AtlasIndexBackingStoreBase) for images
398  * - the macro fastuidraw_fetch_data(B) to fetch the B'th block from the data store buffer
399  * (PainterDraw::m_store), return as uvec4. To get floating point data use, the GLSL
400  * built-in function uintBitsToFloat().
401  * - the macro fastuidraw_fetch_glyph_data(B) to read the B'th value from the glyph data
402  * (GlyphAtlasBackingStoreBase), return as uint. To get floating point data, use the GLSL
403  * built-in function uintBitsToFloat().
404  * - the macro fastuidraw_colorStopFetch(x, L) to retrieve the color stop value at location x of layer L
405  * - vec2 fastuidraw_viewport_pixels the viewport dimensions in pixels
406  * - vec2 fastuidraw_viewport_recip_pixels reciprocal of fastuidraw_viewport_pixels
407  * - vec2 fastuidraw_viewport_recip_pixels_magnitude euclidean length of fastuidraw_viewport_recip_pixels
408  *
409  * For both stages, the value of the argument of shader_data_offset is which block into the data
410  * store (PainterDraw::m_store) of the custom shader data. Do
411  * \code
412  * fastuidraw_fetch_data(shader_data_offset)
413  * \endcode
414  * to read the raw bits of the data. The type returned by the macro fastuidraw_fetch_data()
415  *
416  * Use the GLSL built-in uintBitsToFloat() to covert the uint bit-value to float
417  * and just cast int() to get the value as an integer.
418  *
419  * Also, if one defines macros in any of the passed ShaderSource objects,
420  * those macros MUST be undefined at the end. In addition, if one
421  * has local helper functions, to avoid global name collision, those
422  * function names should be wrapped in the macro FASTUIDRAW_LOCAL()
423  * to make sure that the function is given a unique global name within
424  * the uber-shader.
425  *
426  * Lastly, one can use the classes \ref shader_unpack_value
427  * and \ref shader_unpack_value_set to generate shader code
428  * to unpack values from the data in the data store buffer.
429  * That machine generated code uses the macro fastuidraw_fetch_data().
430  */
431  class PainterItemShaderGLSL:public PainterItemShader
432  {
433  public:
434  /*!
435  * Ctor.
436  * \param puses_discard set to true if and only if the shader code
437  * will use discard. Discard should be used
438  * in the GLSL code via the macro FASTUIDRAW_DISCARD.
