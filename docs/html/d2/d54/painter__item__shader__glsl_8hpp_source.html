33  /*!
34  * \brief
35  * A PainterItemCoverageShaderGLSL is a collection of GLSL source code
36  * fragments for a \ref PainterShaderRegistrarGLSL.
37  *
38  * The vertex shader code needs to implement the function:
39  * \code
40  * void
41  * fastuidraw_gl_vert_main(in uint sub_shader,
42  * in uvec4 attrib0,
43  * in uvec4 attrib1,
44  * in uvec4 attrib2,
45  * in uint shader_data_offset,
46  * out vec3 clip_p)
47  * \endcode
48  * where
49  * - sub_shader corresponds to PainterItemCoverageShader::sub_shader()
50  * - attrib0 corresponds to PainterAttribute::m_attrib0,
51  * - attrib1 corresponds to PainterAttribute::m_attrib1,
52  * - attrib2 corresponds to PainterAttribute::m_attrib2 and
53  * - shader_data_offset is what block in the data store for
54  * the data packed by PainterItemCoverageShaderData::pack_data()
55  * of the PainterItemCoverageShaderData in the \ref Painter call;
56  * use the macro fastuidraw_fetch_data() to read the data.
57  *
58  * The output clip_p is to hold the clip-coordinate of the vertex.
59  *
60  * The fragment shader code needs to implement the function:
61  * \code
62  * float
63  * fastuidraw_gl_frag_main(in uint sub_shader,
64  * in uint shader_data_offset)
65  * \endcode
66  *
67  * which returns the value to write to the coverage buffer
68  * from the fragment for the item.
69  *
70  * Available to only the vertex shader are the following:
71  * - mat3 fastuidraw_item_matrix (the 3x3 matrix from item coordinate to clip coordinates)
72  *
73  * Available to both the vertex and fragment shader are the following:
74  * - sampler2DArray fastuidraw_imageAtlasLinear the color texels (AtlasColorBackingStoreBase) for images unfiltered
75  * - sampler2DArray fastuidraw_imageAtlasLinearFiltered the color texels (AtlasColorBackingStoreBase) for images bilinearly filtered
76  * - usampler2DArray fastuidraw_imageIndexAtlas the texels of the index atlas (AtlasIndexBackingStoreBase) for images
77  * - the macro fastuidraw_fetch_data(B) to fetch the B'th block from the data store buffer
78  * (PainterDraw::m_store), return as uvec4. To get floating point data use, the GLSL
79  * built-in function uintBitsToFloat().
80  * - the macro fastuidraw_fetch_glyph_data(B) to read the B'th value from the glyph data
81  * (GlyphAtlasBackingStoreBase), return as uint. To get floating point data, use the GLSL
82  * built-in function uintBitsToFloat().
83  * - the macro fastuidraw_colorStopFetch(x, L) to retrieve the color stop value at location x of layer L
84  * - vec2 fastuidraw_viewport_pixels the viewport dimensions in pixels
85  * - vec2 fastuidraw_viewport_recip_pixels reciprocal of fastuidraw_viewport_pixels
86  * - vec2 fastuidraw_viewport_recip_pixels_magnitude euclidean length of fastuidraw_viewport_recip_pixels
87  *
88  * For both stages, the value of the argument of shader_data_offset is which block into the data
89  * store (PainterDraw::m_store) of the custom shader data. Do
90  * \code
91  * fastuidraw_fetch_data(shader_data_offset)
92  * \endcode
93  * to read the raw bits of the data. The type returned by the macro fastuidraw_fetch_data()
94  *
95  * Use the GLSL built-in uintBitsToFloat() to covert the uint bit-value to float
96  * and just cast int() to get the value as an integer.
97  *
98  * Also, if one defines macros in any of the passed ShaderSource objects,
99  * those macros MUST be undefined at the end. In addition, if one
100  * has local helper functions, to avoid global name collision, those
101  * function names should be wrapped in the macro FASTUIDRAW_LOCAL()
102  * to make sure that the function is given a unique global name within
103  * the uber-shader.
104  *
105  * Lastly, one can use the class \ref UnpackSourceGenerator to generate
106  * shader code to unpack values from the data in the data store buffer.
107  * That machine generated code uses the macro fastuidraw_fetch_data().
108  */
