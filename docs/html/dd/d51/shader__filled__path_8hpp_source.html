43 /*!
44  * \brief
45  * A ShaderFilledPath represents a path that is drawn as a rectangle
46  * where the fragment shader acting on the rectangle performs the
47  * coverage computation of each pixel. Generally speaking, one
48  * should only fill paths via a ShaderFilledPath when a filled
49  * path when drawn has very high density of edges.
50  */
