Detailed Description

A PainterBrush defines a brush for painting via Painter.

The brush applies, in the following order:

  1. a constant color, specified by pen()
  2. optionally applies an image, specified by image() or sub_image() also see no_image()
  3. optionally applies a linear or radial gradient, see linear_gradient() and radial_gradient() also see no_gradient().

An item shader's vertex stage provides the coordinate fed to the brush. That coordinate is procesed in the following order before it is fed to the image and gradient:

  1. an optional 2x2 matrix is applied specified by transformation_matrix(), also see no_transformation_matrix() and transformation().
  2. an optional translation is applied specified by transformation_translate(), also see no_transformation_translation() and transformation().
  3. an optional repeat window is applied specified by repeat_window(), also see no_repeat_window().

◆ image_offset_t

enum fastuidraw::PainterBrush::image_offset_t

Offsets for image data packing.

The number of index look ups is recorded in PainterBrush::shader(). The ratio of the size of the image to the size of the master index is given by pow(I, Image::number_index_lookups). where I is given by ImageAtlas::index_tile_size().

Enumerator
image_size_xy_offset 

Width and height of the image (Image::dimensions()) encoded in a single uint32. The bits are packed as according to image_size_encoding

image_start_xy_offset 

top left corner of start of image to use (for example using the entire image would be (0,0)). Both x and y start values are encoded into a single uint32. Encoding is the same as image_size_xy_offset, see image_size_encoding

image_atlas_location_xyz_offset 

Location of image (Image::master_index_tile()) in the image atlas is encoded in a single uint32. The bits are packed as according to image_atlas_location_encoding. If the image is not of type Image::on_atlas, gives the high 32-bits of Image::handle().

image_bindless_handle_hi_offset 

Alias for image_atlas_location_xyz_offset to be used when packing an image whose type is not Image::on_atlas.

image_slack_number_lookups_offset 

Holds the amount of slack in the image (see Image::slack()) and the number of index looks ups (Image::number_index_lookups()) with bits packed as according to image_slack_number_lookups_encoding. If the image is not of type Image::on_atlas, gives the low 32-bits of Image::handle().

image_bindless_handle_low_offset 

Alias for image_slack_number_lookups_offset to be used when packing an image whose type is not Image::on_atlas.

image_data_size 

Number of elements packed for image support for a brush.

◆ shader()

uint32_t fastuidraw::PainterBrush::shader ( void  ) const

Returns the brush shader ID which when tested against the bit masks from shader_masks tells what features are active in the brush. The shader is decoded as follows:

  • The value given by
    unpack_bits(image_filter_bit0, image_filter_num_bits, shader())
    is non-zero if an image is present and when is non-zero the value's meaning is enumerated by image_filter
  • If shader() & gradient_mask is non-zero, then a gradient is applied. The gradient is a linear gradient if shader() & radial_gradient_mask is zero and a radial gradient otherwise.
  • If shader() & radial_gradient_mask is non-zero, then a radial gradient is applied. Note that if shader() & radial_gradient_mask is non-zero, then shader() & gradient_mask is also non-zero.
  • If shader() & gradient_repeat_mask then the gradient is repeated instead of clamped. Note that if shader() & gradient_repeat_mask is non-zero, then shader() & gradient_mask is also non-zero.
  • If shader() & repeat_window_mask is non-zero, then a repeat window is applied to the brush.
  • If shader() & transformation_translation_mask is non-zero, then a translation is applied to the brush.
  • If shader() & transformation_matrix_mask is non-zero, then a 2x2 matrix is applied to the brush.
