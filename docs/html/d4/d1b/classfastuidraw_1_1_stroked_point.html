Detailed Description

A StrokedPoint holds the data for a point of stroking where all segments are line segments (coming from curve tessellation). The upshot is that the fragment shader does NOT perform any coverage computation for non-dashed stroking. In addition, the data is so that changing the stroking width or miter limit does not change the stroking data.

enum fastuidraw::StrokedPoint::offset_type_t

Enumeration for specifing the point type which in turn determines the meaning of the fields m_pre_offset and m_auxiliary_offset.

Enumerator
offset_sub_edge 

The point is for an edge of the path, point signifies the start or end of a sub-edge (quad) of drawing an edge. The meanings of m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset the normal vector to the edge in which move the point by when stroking
  • m_auxiliary_offset when added to m_position, gives the position of the point on the other side of the edge.
offset_shared_with_edge 

The point is at a position that has the same value as point on an edge but the point itself is part of a cap or join. The meanings of the members m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset the normal vector to the edge in which move the point by the stroking width when stroking; this vector can be (0, 0).
  • m_auxiliary_offset unused (set to (0, 0))
offset_rounded_join 

The point is for a boundary point of a rounded join of the path. The meanings of the members m_pre_offset and and m_auxiliary_offset are:

  • m_pre_offset the .x() component holds the unit normal vector between the join point and the edge going into the join. The .y() component holds the unit normal vector between the join point and the edge leaving the join. The packing is that the x-coordinate value is given and the y-coordinate magnitude is . If the bit normal0_y_sign_bit is up, then the y-coordinate for the normal vector of going into the join, then the y-value is negative. If the bit normal1_y_sign_bit is up, then the y-coordinate for the normal vector of leaving the join, then the y-value is negative.
  • m_auxiliary_offset the .x() component gives an interpolation in the range [0, 1] to interpolate between the normal vectors packed in m_pre_offset. The .y() value gives the normal vector directly but packed (as in m_pre_offset) where the y-coordinate sign is negative if the bit sin_sign_mask is up.
offset_miter_clip_join 

Point type for miter-clip join point whose position depends on the stroking radius and the miter-limit. The meanings of m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset gives the unit normal vector of the edge going into the join
  • m_auxiliary_offset gives the unit normal vector of the edge leaving the join
offset_miter_bevel_join 

Point type for miter-bevel join point whose position depends on the stroking radius and the miter-limit. The meanings of m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset gives the unit normal vector of the edge going into the join
  • m_auxiliary_offset gives the unit normal vector of the edge leaving the join
offset_miter_join 

Point type for miter join whose position position depends on the stroking radius and the miter-limit. The meanings of m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset gives the unit normal vector of the edge going into the join
  • m_auxiliary_offset gives the unit normal vector of the edge leaving the join
offset_rounded_cap 

The point is for a boundary point of a rounded cap of the path. The meanings of m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset is the normal vector to the path to start drawing the rounded cap
  • m_auxiliary_offset gives the the unit vector (cos, sin) of the angle to make with the vector given by m_pre_offset
offset_square_cap 

The point is for a boundary point of a square cap of the path, the meanings of m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset is the normal vector to the path by which to move the point
  • m_auxiliary_offset is the tangent vector to the path by which to move the point
offset_adjustable_cap 

The point is a point of an adjustable cap. These points are for dashed stroking with caps; they contain data to allow one from a vertex shader to extend or shrink the cap area correctly to implement dashed stroking. The meanings of m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset is the normal vector to the path by which to move the point; this value can be (0, 0) to indicate to not move perpindicular to the path
  • m_auxiliary_offset is the tangent vector to the path by which to move the point; this value can be (0, 0) to indicate to not move parallel to the path
offset_flat_cap 

The point is a point of a flat cap. These points are for doing anti-aliasing at the starting and ending of an open contour. The meanings of m_pre_offset and m_auxiliary_offset are:

  • m_pre_offset is the normal vector to the path by which to move the point; this value can be (0, 0) to indicate to not move perpindicular to the path
  • m_auxiliary_offset is the tangent vector to the path by which to move the point; this value can be (0, 0) to indicate to not move parallel to the path
number_offset_types 

Number different point types with respect to rendering

void fastuidraw::StrokedPoint::pack_point ( PainterAttribute *  dst) const

Pack the data of this StrokedPoint into a Painter Attribute. The packing is as follows:

  • PainterAttribute::m_attrib0 .xy -> m_position (float)
  • PainterAttribute::m_attrib0 .zw -> m_pre_offset (float)
  • PainterAttribute::m_attrib1 .x -> m_distance_from_edge_start (float)
  • PainterAttribute::m_attrib1 .y -> m_distance_from_contour_start (float)
  • PainterAttribute::m_attrib1 .zw -> m_auxiliary_offset (float)
  • PainterAttribute::m_attrib2 .x -> m_packed_data (uint)
  • PainterAttribute::m_attrib2 .y -> m_edge_length (float)
  • PainterAttribute::m_attrib2 .z -> m_contour_length (float)
  • PainterAttribute::m_attrib2 .w (free)
